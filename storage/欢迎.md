# 欢迎使用 nas-md

nas-md 是一个轻量级的 Markdown 笔记服务，专为 NAS 和自托管场景设计。

## 快速上手

### 打开文件

在左侧边栏选择一个挂载目录，点击 `.md` 文件即可打开编辑。

### 创建文件

在可读写的目录下，点击目录名右侧的 **+** 按钮即可新建文件夹或 Markdown 文件。

### 编辑模式

顶部工具栏可切换三种模式：

- **所见即所得（WYSIWYG）** —— 像 Word 一样直接编辑排版，推荐日常使用
- **即时渲染（IR）** —— 输入 Markdown 标记后立即渲染，适合习惯 Markdown 语法的用户
- **分屏预览（SV）** —— 左侧写 Markdown 源码，右侧实时预览渲染结果

### 保存

- **自动保存**默认开启，编辑后自动保存到服务器
- 也可手动点击「保存」按钮或按 `Ctrl+S`

## 挂载目录

### 宿主机目录

Docker 部署时在 `volumes` 中挂载的宿主机目录会出现在侧边栏：

- **私有目录** —— 仅 Admin 可见可读写
- **公开目录** —— 所有用户可见可读写（通过 `PUBLIC_MOUNTS` 环境变量配置）

### 浏览器挂载

在欢迎页点击「浏览…」按钮，选择本地目录即可挂载：

- HTTPS 环境下支持读写
- HTTP 环境下仅只读

## 图片引用

Markdown 中使用相对路径引用的图片会自动解析显示。例如：

```markdown
![示例图片](photo.png)
```

只要 `photo.png` 和当前 `.md` 文件在同一目录下，就能正确显示。

## 快捷键

| 快捷键 | 操作 |
|--------|------|
| `Ctrl+K` | 搜索文件 |
| `Ctrl+S` | 保存 |
| `Ctrl+N` | 新建笔记 |
| `Ctrl+B` | **粗体** |
| `Ctrl+I` | *斜体* |
| `Ctrl+~` | 切换侧边栏 |

## 更多功能

- **全文搜索** —— 侧边栏搜索框，毫秒级检索
- **暗色模式** —— 顶部月亮图标一键切换
- **双向链接** —— 使用 `[[文件名]]` 语法链接笔记，自动追踪反向链接
- **知识图谱** —— 可视化笔记之间的链接关系
- **文件操作** —— 重命名、移动、复制、删除，支持跨挂载点操作
- **锚点跳转** —— 使用 `[跳转](#标题)` 格式跳转到文章内标题位置

## API 参考

所有 API 基于 REST 风格，返回 JSON 格式数据。普通用户只能访问公开挂载点，Admin 用户（请求头 `X-Admin: 1`）可访问所有挂载点。

### 通用信息

- **基础 URL**：`http(s)://<host>:<port>/api`
- **Admin 标识**：请求头添加 `X-Admin: 1`
- **会话标识**：Cookie 中的 `session_id`

---

### 系统接口

#### 健康检查

```
GET /api/health
```

返回 `{"status": "ok"}`

#### 运行配置

```
GET /api/config
```

返回 `{"docker_mode": true|false}`

---

### 挂载点管理

#### 列出挂载点

```
GET /api/mounts
```

返回当前用户可见的挂载点列表。Admin 可见所有挂载点，普通用户仅见公开和自己的挂载点。

**响应示例**：

```json
[
  {
    "id": "docs",
    "name": "文档",
    "path": "/mnt/docs",
    "readonly": false,
    "public": true,
    "host": true
  }
]
```

#### 列出公开挂载点

```
GET /api/mounts/public
```

返回所有公开挂载点及访客自己的挂载点。

#### 添加挂载点（Admin）

```
POST /api/mounts
```

**请求体**：

```json
{
  "name": "共享文档",
  "path": "/mnt/docs",
  "readonly": false,
  "public": true
}
```

#### 更新挂载点属性（Admin）

```
PUT /api/mounts/{id}
```

**请求体**：

```json
{
  "name": "新名称",
  "readonly": true,
  "public": false
}
```

#### 删除挂载点（Admin）

```
DELETE /api/mounts/{id}
```

### 文件操作

#### 获取目录树

```
GET /api/mounts/{id}/tree?path=/
```

返回指定路径下的目录结构（单层）。

**参数**：

| 参数 | 说明 |
|------|------|
| `path` | 目录路径，默认 `/` |

#### 递归获取目录树

```
GET /api/mounts/{id}/tree-recursive?path=/
```

返回完整的多层目录结构。

#### 读取文件

```
GET /api/mounts/{id}/file?path=/notes/readme.md
```

返回文件内容（纯文本）。

#### 写入文件

```
PUT /api/mounts/{id}/file?path=/notes/readme.md
```

**请求体**：文件内容（纯文本）

只读挂载点返回 `403`。

#### 创建文件或文件夹

```
POST /api/mounts/{id}/create?path=/notes/new.md&type=file
POST /api/mounts/{id}/create?path=/notes/subdir&type=folder
```

**参数**：

| 参数 | 说明 |
|------|------|
| `path` | 新建路径 |
| `type` | `file` 或 `folder` |

#### 重命名

```
PUT /api/mounts/{id}/rename?path=/notes/old.md&name=new.md
```

**参数**：

| 参数 | 说明 |
|------|------|
| `path` | 原文件路径 |
| `name` | 新文件名 |

#### 创建目录

```
PUT /api/mounts/{id}/mkdir?path=/notes/new-folder
```

#### 移动文件

```
POST /api/mounts/{id}/move?src=/notes/a.md&dst=/archive/a.md
```

#### 复制文件

```
POST /api/mounts/{id}/copy?src=/notes/a.md&dst=/backup/a.md
```

#### 删除文件

```
DELETE /api/mounts/{id}/file?path=/notes/old.md
```

#### 跨挂载点移动（Admin）

```
POST /api/cross-mount-move
```

**请求体**：

```json
{
  "src_mount_id": "docs",
  "src_path": "/notes/a.md",
  "dst_mount_id": "archive",
  "dst_path": "/notes/a.md"
}
```

#### 跨挂载点复制（Admin）

```
POST /api/cross-mount-copy
```

请求体格式同上。

### 搜索与查询

#### 全文搜索

```
GET /api/search?q=关键词
```

搜索所有可见挂载点中的 Markdown 文件内容。

**参数**：

| 参数 | 说明 |
|------|------|
| `q` | 搜索关键词 |

**响应示例**：

```json
[
  {
    "path": "/notes/readme.md",
    "filename": "readme.md",
    "title": "项目说明",
    "snippet": "...包含关键词的上下文片段...",
    "mount_id": "docs"
  }
]
```

#### 结构化查询

```
GET /api/query?type=recent&limit=10
GET /api/query?type=by-tag&tag=重要
```

#### 反向链接

```
GET /api/backlinks?path=/notes/readme.md&mount_id=docs
```

返回链接到指定文件的所有文件列表。

#### 查找文件路径

```
GET /api/find-path?name=readme.md
```

在所有挂载点中查找指定文件名的路径。

---

### 数据分析

#### 统计信息

```
GET /api/stats
```

返回文件数量、目录数量等统计。

#### 知识图谱

```
GET /api/graph
```

返回笔记之间的链接关系图数据。

#### 标签列表

```
GET /api/tags
```

返回所有标签及其出现次数。

#### 孤立文件

```
GET /api/orphans
```

返回没有被任何其他文件链接的文件列表。

---

### 同步接口

#### 同步状态

```
GET /api/sync/status?mount_id=docs
```

#### 增量同步

```
POST /api/sync?mount_id=docs
```
